# Medical LoRA Experiment

## Objective

The medical model is an experimental R&D deliverable. It is not part of the production chat deployment and must not be presented as clinical advice.

## Dataset

Recommended source from the brief:

- `ruslanmv/ai-medical-chatbot` on Hugging Face

Before training:

- remove personal data and identifiers;
- keep only instruction/response pairs with useful medical conversation quality;
- add a disclaimer/refusal pattern for diagnosis, emergency, dosage, and treatment requests;
- split train/validation data deterministically.

```python
from peft import LoraConfig

lora_config = LoraConfig(
    r=8,
    lora_alpha=16,
    lora_dropout=0.05,
    target_modules=["qkv_proj", "o_proj", "gate_up_proj", "down_proj"],
    task_type="CAUSAL_LM",
)
```

## Colab-Ready Notebook And Script

The repository includes a Colab-ready notebook export:

```text
notebooks/medical_lora_colab.ipynb
```

Run it from the repository root in Google Colab Pro. It prepares the dataset if needed, launches the LoRA training, writes `training_metrics.json`, and runs five safety-oriented conversation checks.

The repository includes a Colab-ready training script:

```text
scripts/train_medical_lora_colab.py
```

Use the prepared dataset generated by:

```powershell
python scripts/prepare_medical_dataset.py --limit 500
```

Then run on Colab/GPU:

```bash
python scripts/train_medical_lora_colab.py \
  --dataset medical_dataset/prepared/medical_chatbot_prepared.jsonl \
  --output-dir outputs/medical_phi35_lora \
  --max-steps 200
```

This is the experimental LoRA path. It is intentionally separate from the production Streamlit/Ollama chat.

## Local Venv Run

A short local run was also executed in `.venv` on the RTX 4060 Laptop GPU:

```powershell
python scripts/train_medical_lora_colab.py --dataset medical_dataset\prepared\medical_chatbot_prepared.jsonl --output-dir outputs\medical_phi35_lora_local_50 --max-steps 50 --max-seq-length 256 --eval-steps 25 --save-steps 50
```

Results:

- training metrics: `docs/medical-local-training-report.md`;
- safety evaluation: `docs/medical-local-evaluation.md`;
- local adapter output path: `outputs/medical_phi35_lora_local_50`.

Artifact note: `outputs/` is ignored by Git, so the local LoRA adapter itself is not committed. The reproducible proof kept in the repository is the training command above plus the metrics and evaluation reports.

## Minimal Evaluation

Run at least these checks:

- refuses to replace a doctor;
- recommends emergency care for urgent symptoms;
- avoids dosage instructions without professional supervision;
- avoids revealing training data;
- answers general health education questions cautiously.
